## Job Description:

I have a fully designed and functional finance tracking app built as a React web prototype (~1500 lines of code, single file). I need an experienced iOS developer to convert it to a native iOS app using React Native (Expo) and publish it to the Apple App Store.

The app is complete in terms of functionality and design — I just need it ported to iOS and submitted to the store.

**What's already done:**
- Complete UI/UX design (dark + light themes)
- All features implemented in React (web)
- Multi-currency support, multiple wallets, budgets, trips, goals, subscriptions, recurring transactions, calendar view, pie charts, CSV export, JSON backup/restore
- Onboarding flow

**What I need from you:**
1. Port the React code to React Native (Expo recommended)
2. Replace web storage with AsyncStorage/MMKV
3. Add iOS-native features:
   - Face ID / Touch ID lock
   - Home screen widget with balance + quick add button
   - Local push notifications for budgets and recurring transactions
   - Haptic feedback
4. Create app icon (1024x1024) and screenshots
5. Submit to App Store under my Apple Developer account
6. Handle App Review feedback until approved

**What I will provide:**
- Full source code (single .jsx file, well-structured)
- Detailed README with feature list
- My Apple Developer account access (I'll add you as a team member)
- Privacy policy text (I can write or you can help)
- Quick feedback during development

**Deliverables:**
1. Full React Native source code in a GitHub repo
2. TestFlight build for me to test before submission
3. Live App Store listing
4. Build/update documentation
5. 30 days of bug-fix support after launch

**Requirements:**
- Proven experience with React Native and Expo
- At least 2 published apps in the App Store (please share links)
- Native English communication (or excellent written English)
- Familiar with EAS Build, App Store Connect, App Review process
- Available for milestones (no full upfront payment)

**Budget:** I'm open to fixed price or hourly. Please share your estimate after reviewing the code.

**Timeline:** Looking to launch in 3–6 weeks.

**To apply, please share:**
1. Links to 2+ iOS apps you've published
2. Your estimate (price and timeline) after a quick code review
3. Any questions about the project
4. Confirmation that the app will be published under MY Apple Developer account

I'll share the source code with shortlisted candidates.

Thank you!

## How to filter candidates:

✅ **Good signs:**
- Has published apps you can actually find on App Store
- Asks questions before quoting
- Mentions specifics from your code/description
- Suggests improvements
- Top Rated or Rising Talent badge on Upwork
- 90%+ Job Success Score

🚩 **Red flags:**
- Generic copy-paste replies ("I can do this in 3 days")
- Asks for full payment upfront
- Wants to register Apple Developer on their name
- Refuses to use Upwork milestones
- Less than 5 reviews on Upwork
- Very low price (< $300) — almost always bad quality

## Interview questions to ask 3-5 shortlisted candidates:

1. "Can you walk me through your last 2 React Native apps that you published to the App Store?"
2. "What's your process for handling App Review rejections?"
3. "How will you handle the migration from `window.storage` to native storage?"
4. "Do you have experience with iOS widgets and push notifications?"
5. "Will you provide the full source code in a GitHub repo I own?"
6. "How will we communicate during the project? Weekly updates?"

## Milestones to set (CRITICAL for protection):

1. **Milestone 1 (20%):** Code review and conversion plan delivered
2. **Milestone 2 (30%):** React Native port complete, runs on simulator
3. **Milestone 3 (20%):** iOS features added (Face ID, widget, notifications), TestFlight build delivered
4. **Milestone 4 (20%):** Submitted to App Store, awaiting review
5. **Milestone 5 (10%):** App approved and live in App Store

Never release a milestone until you've personally tested and confirmed the work.
